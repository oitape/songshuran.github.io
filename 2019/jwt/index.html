JSON WEB TOKEN

什么是JWT

JSON网络令牌（JWT）是JSON的基于开放标准（RFC 7519），用于创建访问令牌该断言权利要求中的一些数字。例如，服务器可以生成具有声明“以管理员身份登录”并将其提供给客户端的令牌。然后，客户端可以使用该令牌来证明它以管理员身份登录。令牌由一方的私钥（通常是服务器的）签名，以便双方（另一方已经通过一些合适且值得信赖的方式拥有相应的公钥）能够验证令牌是否合法。该令牌被设计为紧凑，[2] URL -safe，[3]和可用特别是在web浏览器 的单点登录（SSO）上下文。JWT声明通常可用于在身份提供者和服务提供者之间传递经过身份验证的用户的身份，或者业务流程所要求的任何其他类型的声明

JWT的结构

如下所示的一个JWT生成的字符串

eyJhbGciOiJIUzUxMiJ9.eyJzdWIiOiJ7XCJ1c2VybmFtZVwiOlwicmFveml3ZW5cIixcImFsaWFzXCI6XCLppbblrZDmlodcIixcImF1dGhlbnRpY2F0ZVwiOnRydWV9IiwiZXhwIjoxNTU1NDgyMDEwfQ.t3RvzrqTwfru1H1q90S9so_2EUlqppf2xIJUEztiPr4G2e6mfW_wClceQWVP-5xX7T_4nWHP5PK8e5_ea26MLQ

细心的会发现该串是由三段组成，用. 英文句点隔开

  • 第一部分：header 头部 ——> 这部分指明签名算法，描述了该LWT的最基本信息

  • 第二部分：payload 载荷 ——> 业务中有效的信息一般存在这里，包含有三部分

    • 标准声明 官方提供的字段
      iss: jwt签发者
      sub: jwt所面向的用户
      aud: 接收jwt的一方
      exp: jwt的过期时间，这个过期时间必须要大于签发时间
      nbf: 定义在什么时间之前，该jwt都是不可用的.
      iat: jwt的签发时间
      jti: jwt的唯一身份标识，主要用来作为一次性token,从而回避重放攻击。
    • 公共声明
      公共的声明可以添加任何的信息，一般添加用户的相关信息或其他业务需要的必要信息。不建议添加敏感信息，因为该部分在客户端可解密。
    • 私有声明
      私有声明是提供者和消费者所共同定义的声明，不建议存放敏感信息

  • 第三部分：signature 签名 ——> 对前两部分的签名，防止数据篡改。
    需要指定密钥，这个密钥保存在服务端，不能泄露给其他端。然后会按照header中指定算法进行签名。算出签名以后，把 Header、Payload、Signature 拼成一个字符串，每个部分之间用（.）分隔。就生成一个可以用作鉴权的token了
